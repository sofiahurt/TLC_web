Crea una aplicación web con Node.js y Bootstrap 5 que sea completamente 
responsive. La base de datos es SQL Server y debes usar el paquete `mssql` 
para la conexión. Usa Express.js como framework, express-session para 
sesiones, y bcrypt NO se usará — las contraseñas se almacenan y comparan 
en texto plano.

## ESTRUCTURA DE CARPETAS
/app
/config         → conexión a SQL Server
/routes         → una ruta por módulo
/views          → archivos EJS por módulo
/partials     → header, sidebar, footer
/public
/css
/js
app.js

## BASE DE DATOS — SQL Server
Esquema: Empresa2 (excepto UsuariosWeb que es dbo)
Cadena de conexión configurable en /config/db.js con variables:
- server, database, user, password, port

## LOGIN

- Ruta GET /login → muestra formulario (usuario + contraseña)
- Ruta POST /login → busca en UsuariosWeb por NOMBRE, compara PASSWORD 
  en texto plano, verifica ACTIVO = 1
- Si es válido: guarda en sesión { id, nombre, apellido, serie } 
  y redirige a /dashboard
- Si falla: muestra mensaje de error en el mismo formulario
- Todas las rutas excepto /login requieren middleware de autenticación 
  que verifique req.session.usuario, si no existe redirige a /login
- Ruta GET /logout → destruye sesión y redirige a /login

## BROWSE (listado de registros)

Aplica estas reglas a TODOS los módulos:

1. **Paginación**: máximo 20 registros por página. Botones Anterior / 
   Siguiente. Indicador "Página X de Y — Z registros totales".

2. **Ordenamiento**: clic en cualquier encabezado de columna ordena 
   ascendente; segundo clic ordena descendente. Indicador visual 
   de columna activa y dirección.

3. **Buscador**: en la parte superior izquierda del browse. Tiene dos 
   controles: un dropdown para seleccionar la columna de búsqueda, 
   y un input de texto. Al teclear (evento oninput) realiza búsqueda 
   automática sin botón, usando LIKE '%valor%' en la columna 
   seleccionada, sin importar posición. La búsqueda resetea a página 1.

4. **Botones CRUD**: agrupados a la derecha, fuera de la tabla:
   - [+ Agregar] → abre popup de alta
   - [✏ Editar] → abre popup de edición con el registro seleccionado
   - [🗑 Eliminar] → muestra confirmación, luego DELETE físico
   - Editar y Eliminar se activan solo cuando hay un registro 
     seleccionado (clic en fila lo resalta).

5. **Columnas a mostrar por tabla** (mostrar las más relevantes, 
   no todas):
   - Camiones: ID_CAMION, MARCA, PLACA, TIPO, OPERADOR, DESCRIPCION
   - Ciudades: ID_CIUDAD, CIUDAD
   - Clientes: ID_CLIENTE, NOMBRECOMUN, RFC, CIUDAD, ESTADO, TIPOCLIENTE
   - ColCP: ID_COLONIA, COLONIA, CP
   - Contacto: ID_CLIENTE, ID_CONTACTO, CONTACTO, EMAIL, AREA, TELEFONO1
   - DieselPre: ID_DIESELPRECIO, FECHA, PRECIOLITRO, IEPS
   - DomCarDes: ID_CLIENTE, ID_DOMICILIO, DESCRIPCION, TIPO, CIUDAD, ESTADO
   - Operadores: ID_OPERADOR, OPERADOR, RFC, CIUDAD, STATUS
   - ProdServ: Clave, ProdServ, Descripcion_SAT, Unidad, Precio
   - Tarifas: ID_TARIFA, CDORIGEN, CDDESTINO, FLETE, NOMBRECOMUNCLI, 
               NOMBRECOMUNTRANS, COSTO, VENTA
   - TarKilomts: Id_TarKmts, Fecha, PrecioKmt
   - TipoFlete: ID_TIPOFLETE, FLETE
   - Transport: ID_TRANSPORTISTA, NOMBRECOMUN, RFC, CIUDAD, CP

## FORMULARIOS (popups modal Bootstrap)

Aplica estas reglas a TODOS los formularios de alta y edición:

1. **Tipo**: modal Bootstrap centrado, tamaño lg o xl según la cantidad 
   de campos.

2. **Layout**: campos organizados en filas de 2 o 3 columnas usando 
   el grid de Bootstrap. Todos los labels alineados y los inputs 
   con ancho proporcional al tipo de dato (un ID corto no ocupa 
   lo mismo que una dirección).

3. **Campos con lookup** (relación a otra tabla): mostrar el campo 
   descriptivo como texto readonly + botón de lupa [🔍] al lado. 
   Al hacer clic abre un segundo modal (lookup) con:
   - Browse de la tabla relacionada (máximo 10 registros, paginado)
   - Buscador en la parte superior
   - Al hacer clic en un registro: cierra el lookup, llena el ID 
     (hidden) y el campo descriptivo (readonly) en el formulario padre

4. **Lookups requeridos por tabla**:
   - Camiones → Operadores (ID_OPERADOR / OPERADOR)
   - Clientes → ColCP (ID_COLONIA / COLONIA+CP)
   - DomCarDes → Clientes (ID_CLIENTE / NOMBRECOMUN), 
                  ColCP (ID_COLONIA / COLONIA)
   - Contacto → Clientes (ID_CLIENTE / NOMBRECOMUN)
   - Transport → ColCP (ID_COLONIA / COLONIA)
   - Tarifas → Ciudades origen (ID_CIUDADORIGEN / CIUDAD),
               Ciudades destino (ID_CIUDADDESTINO / CIUDAD),
               TipoFlete (ID_TIPOFLETE / FLETE),
               Transport (ID_TRANSPORTISTA / NOMBRECOMUN),
               Clientes (ID_CLIENTE / NOMBRECOMUN)

5. **Botones del formulario**: [💾 Guardar] y [✖ Cancelar] alineados 
   a la derecha en el footer del modal.

6. **Validaciones mínimas**: campos PK y NOT NULL requeridos. 
   Mostrar mensaje de error inline si fallan.

7. **Al guardar**: INSERT en alta, UPDATE en edición. Cerrar modal 
   y refrescar el browse automáticamente.

## NOTAS TÉCNICAS

- Usa EJS como motor de plantillas
- El partial del sidebar y header se incluyen en todas las vistas
- Los IDs de tipo tinyint deben mostrarse como checkbox en los 
  formularios (0 = desmarcado, 1 = marcado)
- Los campos decimales usan input type="number" con step apropiado
- Los campos date usan input type="date"
- Todos los CHAR de la BD devuelven strings con espacios al final; 
  aplica .trim() siempre antes de mostrar o comparar
- Las queries usan parámetros (@param) nunca concatenación de strings 
  para evitar SQL injection
- El archivo package.json debe incluir todos los paquetes necesarios: 
  express, ejs, mssql, express-session, y cualquier otro que uses
- Incluye un archivo .env.example con las variables de configuración 
  de la BD
- Incluye un README.md con instrucciones de instalación y arranque
